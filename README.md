# 本棚カタログ

Windows PCを本棚サーバーにして、同じLAN内のiPhoneから本のバーコード写真を送る個人用アプリです。

蔵書データ、表紙キャッシュ、アップロード写真はGit管理の対象外です。GitHubへソースを公開・共有しても、ローカルの本棚データは含まれません。

## Windowsアプリ

配布版は `release/Hondana-Catalog-Portable-0.2.0.exe` です。インストールは不要で、EXEをダブルクリックすると独立したウィンドウで起動します。

蔵書、表紙、アップロード画像は `%APPDATA%\HondanaCatalog\data` に保存されます。EXEを更新してもこのフォルダーは上書きされません。Windows Defender ファイアウォールの確認が表示された場合は、「プライベート ネットワーク」を許可してください。

## 開発版の起動

初回だけ、このフォルダーで次を実行します。

```powershell
npm install
```

以後は `start-hondana.cmd` をダブルクリックすると、ビルド後にPCの本棚画面が開きます。起動した黒い画面を閉じるとサーバーも終了します。

## iPhoneから追加

1. PCとiPhoneを同じWi-Fiへ接続します。
2. PC画面左側のQRコードをiPhoneのカメラで読み取ります。
3. 「カメラで撮影」から、本の裏表紙にあるISBNバーコードを大きく撮影します。
4. 読み取りにくい場合は、同じ画面でISBNを手入力できます。

書籍情報はopenBDを優先し、足りない項目をGoogle Booksから補います。クラウドへ本棚データを同期しません。

撮影した画像は最初にiPhone内でISBNを高速解析します。読み取れた場合はPC側の重い画像解析を省略し、難しい写真だけPC側の高精度解析へ回します。

## 本屋での重複チェック

1. 外出前にiPhone画面の「買う前チェック」を開きます。
2. PCと同じLAN内で「出発前に同期」を押します。
3. 「持ち出し保存」を押し、`hondana-pocket.html` をiPhoneの「ファイル」へ保存します。
4. 店頭では保存したHTMLを開き、タイトル・著者・ISBNで所蔵を確認します。

同期した蔵書一覧はブラウザ内にも残ります。持ち出しHTMLには作成時点の蔵書が埋め込まれているため、自宅LANの外で再読み込みしてもPCへの接続は不要です。

## 所蔵・カテゴリ・新刊確認

- 「所有形態」は実本と電子書籍を分けて管理します。実本は保管場所、電子書籍はAmazon Kindle、DMMブックス、楽天Kobo、BOOK☆WALKER、BookLive、ebookjapanなどの媒体を保存できます。
- 電子書籍には作品ページやWeb本棚のURLを保存できます。URL未設定時は、登録したストアの公式サイトへのリンクを表示します。
- 「カテゴリ」は所有形態とは別です。マンガ、小説、技術、ビジネスなどで絞り込めます。
- 「手動追加」では、ISBNのないPDF・EPUB・電子書籍も登録できます。
- 「手動追加」のタイトル欄は、NDLサーチAPIから書籍候補を表示します。候補を選ぶと著者・ISBN・出版社・出版日を補完し、ISBNから表紙も取得します。
- マンガはシリーズ名と巻数を保存できます。対応するISBNではシリーズと巻数を自動入力します。
- 「新刊リスト」では、登録済みの巻より後に刊行された本をシリーズ単位で確認し、ISBNからそのまま本棚へ追加できます。
- 詳細欄の「新刊確認」と新刊リストの「すべて更新」は、国立国会図書館サーチで刊行済みの巻を確認します。
- リマインド日になると、アプリ上部と左側のリマインダーに表示されます。現在はアプリ内通知です。
- 並び替えは新着、名前、作者、シリーズ、保管場所に対応しています。「手動（ドラッグ）」では表紙をドラッグして順番を保存できます。

## 開発用

APIサーバーとフロントエンドを別々に起動する場合は、次をそれぞれ実行します。

```powershell
npm run server
npm run dev
```

本番相当で起動する場合は `npm run app` を使用します。

Windows配布版を作り直す場合は `npm run dist:win` を使用します。
