## 2. 目的

Windows PCを個人用の蔵書保管先とし、PCでは蔵書を整理し、同じプライベートLAN内のiPhoneではISBN画像登録と店頭での重複購入確認を行えるようにする。クラウドアカウントを必須にせず、蔵書・写真・表紙を利用者のPCへ保存する。

## 3. 利用者と環境

- 主利用者は一つの本棚を管理する個人とする。
- PCはWindows x64、スマートフォンはSafariを利用できるiPhoneを主対象とする。
- PCとiPhoneの連携時は、双方が信頼できる同一LANへ接続されていることを前提とする。
- 書誌・表紙・新刊情報の取得時だけインターネット接続を必要とする。

## 4. 機能要件

| ID | 要件 | 受入条件 |
| --- | --- | --- |
| FR-01 | 蔵書登録 | タイトルによる手動登録、ISBN-10/13による自動登録、iPhone画像による登録ができる |
| FR-02 | 書誌補完 | ISBNから書名、著者、出版社、出版日、分類候補、表紙を取得し、取得不能でもISBNだけで登録を継続できる |
| FR-03 | 蔵書編集 | 所有形態、カテゴリ、場所・媒体、読書状態、シリーズ、タグ、メモ、リマインダーを編集できる |
| FR-04 | 蔵書削除 | 確認後に指定した蔵書を削除できる |
| FR-05 | 検索・絞り込み | 書名、著者、出版社、ISBN、シリーズ、場所等を検索し、所有形態・媒体・カテゴリ・状態・出版社・著者・評価・シリーズ有無で絞り込める |
| FR-06 | 並び替え・表示 | 新着、名前、著者、出版社、シリーズ・巻数、場所・媒体、手動順で表示し、任意の棚見出しと本の大きさを設定できる |
| FR-07 | シリーズ追跡 | 同一シリーズを棚で一項目にまとめ、所持巻一覧と、NDL刊行巻から次に登録可能な巻を表示できる |
| FR-08 | リマインダー | 日付と内容を蔵書へ保存し、期限到来をアプリ内に表示できる |
| FR-09 | LAN連携 | PC画面のQRから認証し、iPhoneで画像送信、ISBN入力、蔵書同期ができる |
| FR-10 | 店頭確認 | iPhoneへ最小蔵書データを保存し、オフラインでもタイトル・著者・ISBNで重複確認できる |
| FR-11 | ローカル保存 | 蔵書、アップロード履歴、写真、表紙をPCのユーザーデータ領域へ保存する |
| FR-12 | おすすめ | 読了・評価済み蔵書をもとに、所蔵済みを除いた同著者の候補を表示・登録できる |
| FR-13 | 一括取り込み | 実本・電子書籍の保存先を指定し、最大12枚のスクリーンショットをローカルOCRして書誌候補から選択登録できる。ISBN一覧またはタイトル・著者のTSV/TXTも最大200件処理できる |

## 5. 非機能要件

| ID | 分類 | 要件 |
| --- | --- | --- |
| NFR-01 | セキュリティ | LAN側は起動時トークン、Host、クライアントIP、Originを検査し、APIと画像へ未認証でアクセスさせない |
| NFR-02 | セキュリティ | JSON入力は許可項目・型・長さ・列挙値を検証し、画像は実データ、容量、画素数、静止画形式を検証する |
| NFR-03 | セキュリティ | Electron rendererでNode.js統合を無効化し、context isolationとsandboxを有効化する |
| NFR-04 | 可用性 | JSONの読込・変更・保存を直列化し、一時ファイルから置換して、直前の正常データを`.bak`へ残す |
| NFR-05 | 可用性 | 一つの外部書誌・表紙APIが失敗しても、利用可能な情報またはISBNだけで処理を継続する |
| NFR-06 | 性能 | iPhone内でISBNを先に解析し、PC解析は必要な場合だけ行う。APIはクライアント単位で要求頻度を制限する |
| NFR-07 | 保守性 | HTTP、ユースケース、外部API、永続化、純粋変換を分離し、公開関数はJSDocで契約を示す |
| NFR-08 | プライバシー | 個人蔵書と写真をGitやクラウドへ自動送信しない。外部APIへ送る情報はISBNまたは検索語に限る |
| NFR-09 | 後方互換 | 古い`books.json`の不足項目を起動時に現行形式へ補完する |
| NFR-10 | プライバシー | スクリーンショット画像はWindows内蔵OCRでPC内処理し、抽出した検索語だけを外部書誌APIへ送る |

## 6. 対象外

- 複数利用者のアカウント、権限管理、クラウド同期
- インターネット越しの公開、VPN構築、LAN通信のTLS暗号化
- OS通知、メール通知、書店在庫・価格比較
- 書誌・新刊候補の完全性保証
- 自動世代バックアップ、災害対策、複数PCの同時更新

## 7. 制約と判断基準

- LAN HTTPは同一ネットワークから盗聴され得るため、公共Wi-FiやゲストWi-Fiでは使用しない。
- ルーターのポート転送やインターネット公開は行わない。
- ISBN、書誌、巻数候補には外部データ由来の誤りがあり得る。重要な判断は現物または出版社情報で確認する。
- 詳細な機能は[機能仕様書](FUNCTIONAL-SPEC.md)、防御と残余リスクは[セキュリティ設計](SECURITY.md)を参照する。
