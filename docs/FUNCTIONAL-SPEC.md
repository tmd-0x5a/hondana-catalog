# 機能仕様書

## 1. 蔵書登録

### F-01 手動登録

1. 利用者がタイトルと任意項目を入力する。
2. タイトル入力が2文字以上の場合、NDL候補を最大8件表示する。
3. 候補選択時は書名、著者、出版社、出版日、ISBNをフォームへ反映する。
4. ISBNがある場合は書誌・表紙を取得してから所蔵項目を反映する。ISBNがなければ手動レコードを作る。

タイトルは必須。ISBNを入力した場合はチェックディジットを検証し、ISBN-10はISBN-13へ変換する。

### F-02 iPhone画像登録

1. iPhoneは撮影画像の下半分を優先してZXingで解析する。
2. 読取成功時はISBNと画像を送信する。失敗時は画像だけを送信する。
3. PCは画像実体を検査して保存し、ISBNがなければ複数領域・回転・二値化方式で再解析する。
4. 読取不能時は画像を残し、iPhoneへISBN手動入力を要求する。
5. ISBN確定後、書誌を取得して蔵書へ登録する。

同一ISBNがある場合は新規重複を作らず、書誌だけを更新し、読書状態・所蔵情報・メモ・並び順を維持する。

### F-02A 一括取り込み

1. 利用者は実本または電子書籍を選び、共通の保管場所または電子書籍媒体を指定する。
2. ISBNを1行ずつ、または`タイトル<TAB>著者`、`ISBN<TAB>タイトル<TAB>著者`形式で最大200件入力する。TSV/TXTファイルも同じ形式で読み込める。
3. ISBN行は書誌・表紙取得を行い、タイトル行は手動書誌として保存する。
4. 既存ISBNは重複を作らず指定した所有形態・保存先へ更新する。一行の処理失敗は後続行を止めず、結果一覧へ返す。

## 2. 書誌・表紙

### F-03 書誌統合

- openBDとGoogle Booksを並行して照会する。
- 基本書誌はopenBDを優先し、不足項目をGoogle Booksで補う。
- 両方が失敗または未登録の場合も`ISBN {isbn}`として登録を継続する。
- カテゴリは利用者の保存値、旧マンガ分類、キーワード推定の順で決定する。

### F-04 表紙キャッシュ

- openBD・Google Booksの書誌API候補を優先し、Open Libraryを補助候補としてHTTPS画像を試す。
- 許可ホスト、Content-Type、8MB上限、寸法、画素数、静止画を検査する。
- 120x160未満のプレースホルダーを除外し、最大640x960のWebPへ変換する。
- 一冊の表紙取得失敗は他の蔵書処理を停止させない。
- openBD候補が無効でもGoogle Books、Open Libraryの順に続行し、詳細画面の「表紙」操作でISBNから再取得できる。
- 画面上の画像読込に失敗した場合は、壊れた画像表示をアプリ同梱の既定表紙へ置き換える。

## 3. 蔵書管理

### F-05 編集・削除

- 編集可能項目は書名、著者、出版社、出版日、ページ、読書状態、評価、棚、タグ、メモ、カテゴリ、所有形態、場所・媒体、電子URL、シリーズ、巻数、リマインダーとする。
- ISBNと内部IDは編集APIで変更できない。
- 電子URLは認証情報を含まないHTTPS URLだけ保存できる。
- 削除前にPC画面で確認ダイアログを表示する。

### F-06 検索・絞り込み・並び替え

- 部分一致対象は書名、著者、出版社、ISBN、タグ、カテゴリ、棚、物理場所、電子媒体、シリーズ名。
- 所有形態、カテゴリ、電子媒体、読書状態、出版社、著者、最低評価、シリーズ有無で絞り込む。
- 自動順は新着、名前、著者、出版社、シリーズ・巻数、場所・媒体。手動順はドラッグ結果を全ID配列として保存する。
- 名前・著者・出版社・シリーズ・場所順では対応する棚見出しを表示できる。かなは行、英字・数字は補助見出しへ分け、読みのない漢字は先頭漢字ごとに分ける。見出しは棚上の「仕切り」または表示設定から非表示にできる。
- openBDから取得できるタイトル・著者の読み、または利用者が編集した読みを、名前順・著者順と行見出しへ優先して使う。

### F-06A 本棚表示設定

- 本の幅を72〜132pxで変更し、高さは表紙比率に追従する。
- 同一シリーズを一つの重なりカードへまとめ、クリックで所持巻一覧へ遷移する。
- 手動並び替え中は各巻を直接ドラッグできるよう、シリーズ集約を一時的に解除する。
- 表示設定はPCのlocalStorageへ保存し、蔵書JSONには混在させない。

## 4. シリーズとリマインダー

### F-07 新刊確認

1. マンガかつシリーズ名のある蔵書をシリーズ名の比較キーで集約する。
2. NDLから同名シリーズの巻を取得し、同一巻の重複版を除く。
3. 所持最大巻より大きい最初の巻を「次に登録可能」とする。
4. 最新巻と次巻のISBN・刊行日・NDL URLを同シリーズの全所持巻へ保存する。
5. 新刊リストから次巻ISBNを登録できる。

### F-08 リマインダー

- 蔵書ごとに日付と内容を保存する。
- リマインダー画面は日付がある蔵書を表示し、当日以前を期限到来として強調する。
- OS通知やバックグラウンド通知は行わない。

### F-08A おすすめ

- 読了または評価済みの蔵書から、評価の高い順に最大3人の著者を種として使う。
- NDLサーチAPIを著者名で逐次検索し、所蔵済みISBN、同名タイトル、重複候補を除外して最大8件を表示する。
- 候補の追加は既存のISBN登録処理を使い、登録後は通常の本棚と詳細へ戻る。
- 読書傾向はPC内で選定し、蔵書一覧そのものを外部APIへ送らない。

## 5. LAN連携と持ち出し

### F-09 LAN認証

- 起動時に256bitのランダムトークンを生成し、PCのQRだけへ埋め込む。
- iPhoneの初回GETでトークンをHttpOnly・SameSite=Strict Cookieへ移し、URLから削除する。
- アプリ再起動後はトークンが変わるため、QRから再接続する。
- ループバック接続はデスクトップUIのためトークンを不要とする。

### F-10 店頭用データ

- 「出発前に同期」は最小項目をiPhoneのlocalStorageへ保存する。
- 「持ち出し保存」は外部スクリプトを含まない自己完結HTMLを返す。
- 店頭ではタイトル、著者、ISBNの部分一致とISBN完全一致による登録済み表示を行う。

## 6. エラー表示

- 4xxは利用者が修正できる日本語メッセージを返す。
- 5xxは内部情報を隠し、PCログ確認を案内する。
- ISBN画像読取不能は422とし、保存済みアップロード情報を返して手動補完を可能にする。
- 要求頻度超過は429、LAN認証不足は401、送信元・ネットワーク違反は403とする。
