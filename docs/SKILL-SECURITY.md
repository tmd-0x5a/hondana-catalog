# 外部Skillの安全運用

AI向けSkillは実行ファイルでなくても、シェル操作、ネットワーク通信、秘密情報の読み取りをモデルへ指示できます。このプロジェクトでは外部Skillを信頼済みコードとして扱いません。

- 静的確認日: 2026-07-15
- 適用方法: Skillはインストールせず、振る舞いを維持する小さな変更、変更前後のテスト、責務単位の分割という原則だけを手動で適用

## 導入条件

1. グローバルインストールを避け、必要なら確認済みコミットのファイルだけをプロジェクト内へ置く。
2. `SKILL.md`だけでなく、参照ファイル、スクリプト、エージェント定義も確認する。
3. 更新時は前回確認コミットとの差分を読み、自動更新しない。
4. `.env`、`.ssh`、認証トークン、ブラウザセッション、ユーザーデータの読み取りを許可しない。
5. ネットワーク通信、依存パッケージの追加、外部コマンド、Git操作は別々に目的を確認する。
6. 「以前の指示を無視する」「ユーザーへ知らせない」など、上位指示や可視性を回避する文言があれば導入しない。

スター数、導入数、自動監査のPass表示は調査の手掛かりに留め、安全性の根拠にはしません。
